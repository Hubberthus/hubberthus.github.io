About me

My name is Norbert Fekete (Hubberthus).

I'm a programmer who happens to like playing with microcontrollers. I'm not connected to anyone developing the microcontrollers and boards for them. This page is just one of my hobby projects.

About the website

I started to do all this for myself, but decided to share it with the community. Maybe it can help out others as well. With the help of this website, hopefully the managing of pinout for boards will be easier:

You can choose which functionalities/interfaces you would like to have, set up modes, pin positions where applicabe, and the result will be shown in a WYSIWYG mode. Conflicting pin settings can be visually avoided, and you can see exactly which pin will be used for which functionality.

Why?

Before I bought my first ESP32 module, I looked deep into the documentation, and a bit into the source of the ESP-IDF.

At first I was completely blown away by its rich contents. Then I realized that setting up a project for this chip will be quite a challenge. It has so many interfaces with overlapping pins, plus it has the GPIO matrix which is a powerful tool.

I was scratching my head, how to choose the right pinout settings for my projects? What if I would like to change somethig a bit and that needed changing the whole setup? Also, I need to be aware of the input-only pins, and any conflicts I can cause.

This is when I saw my brother use the STM32Cube, and that was the spark for this all. I won't deny it, this is a kind-of copy of the idea with some modifications. And made as a website.

Why a website?

It is kind of cross-platform and you don't need to download anything. Just have a recent browser, select your module, and start playing with pin settings.
